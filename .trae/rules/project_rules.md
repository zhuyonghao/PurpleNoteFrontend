1. 技术栈
- 使用vite作为脚手架工具
- 使用pinia进行状态管理
- vue router进行路由
- 使用axios发送网络请求并解决跨域访问（CORS）的问题，
- 使用element plus组件库和tailwind，页面美观，功能齐全。

2. 代码规范
- 从接口获取信息，对数据进行适当的处理和格式化，在控制台打印出来便于调试。同时，对可能出现的错误进行捕获和处理，确保程序的健壮性。在开发环境下，使用详细的日志记录，而在生产环境下则使用适当的日志级别。
- 遵循代码复用原则，将通用的组件、样式和业务逻辑进行抽象和封装。建立可复用的组件库，统一的样式系统，以及共享的工具函数，以提高开发效率和代码质量。

3. 项目结构
- 项目结构清晰，按照功能模块进行划分，每个模块有自己的目录和文件。
- 组件、样式和业务逻辑进行了合理的组织和管理。
- 代码规范和命名规范得到了遵守。
- 代码注释清晰，易于理解和维护。

4. 页面设计
- 页面设计参考doc目录下的doc.md，如果你不知道具体的接口怎么开发参考doc目录下的default_OpenAPI.json
